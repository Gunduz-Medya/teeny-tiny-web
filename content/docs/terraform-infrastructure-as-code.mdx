## ðŸ”¹ 6. Using Terraform Variables

Define variables in **`variables.tf`**:

```hcl
variable "instance_type" {
  default = "t2.micro"
}

variable "region" {
  default = "us-east-1"
}
```

Modify **`main.tf`** to use variables:

```hcl
provider "aws" {
  region = var.region
}

resource "aws_instance" "my_instance" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = var.instance_type

  tags = {
    Name = "TerraformInstance"
  }
}
```

âœ… **Now, you can change configurations easily.**

## ðŸ”¹ 7. Managing State with Terraform

Terraform maintains state in a **terraform.tfstate** file.

ðŸ“Œ **Best Practices for Managing State**:
- Store Terraform state in **S3** for shared access.
- Use **Terraform backend configuration** to manage state.

```hcl
terraform {
  backend "s3" {
    bucket = "my-terraform-state"
    key    = "terraform.tfstate"
    region = "us-east-1"
  }
}
```

âœ… **This stores the state file remotely in AWS S3.**
