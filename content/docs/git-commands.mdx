## ðŸ“Œ Git Basics

Below are the most commonly used Git commands for version control, collaboration, and repository management.

### ðŸ”¹ Repository Setup & Initialization

```bash
git init                     # Initialize a new Git repository
git clone <repo-url>         # Clone an existing repository from GitHub/GitLab
git status                   # Check the status of the working directory
git log                      # Show commit history
git remote -v                # Show remote repository URLs
```

### ðŸ”¹ Staging & Committing

```bash
git add <file>               # Add a specific file to staging area
git add .                    # Add all modified and new files to staging
git commit -m "Commit message" # Commit staged files with a message
git commit --amend -m "New message" # Modify the last commit message
git reset HEAD <file>        # Unstage a file without losing changes
```

### ðŸ”¹ Branching & Merging

```bash
git branch                   # List all branches
git branch <new-branch>      # Create a new branch
git checkout <branch>        # Switch to a different branch
git switch <branch>          # Alternative to checkout (Git 2.23+)
git merge <branch>           # Merge a branch into the current branch
git rebase <branch>          # Reapply commits from another branch
git branch -d <branch>       # Delete a branch (only if merged)
git branch -D <branch>       # Force delete a branch
```

### ðŸ”¹ Pushing & Pulling

```bash
git push origin <branch>     # Push changes to the remote repository
git push --set-upstream origin <branch> # Set default upstream branch
git pull origin <branch>     # Pull latest changes from remote
git fetch origin             # Fetch changes without merging
```

### ðŸ”¹ Undoing Changes

```bash
git checkout -- <file>       # Discard local changes in a file
git restore <file>           # Alternative to checkout (Git 2.23+)
git reset --soft HEAD~1      # Undo last commit, keep changes staged
git reset --mixed HEAD~1     # Undo last commit, unstage changes
git reset --hard HEAD~1      # Undo last commit, discard all changes
git revert <commit-hash>     # Revert a specific commit
```

### ðŸ”¹ Working with Tags

```bash
git tag                      # List all tags
git tag -a v1.0 -m "Version 1.0" # Create an annotated tag
git push origin --tags       # Push tags to the remote repository
git checkout tags/v1.0       # Switch to a specific tag
```

### ðŸ”¹ Stashing Changes

```bash
git stash                    # Save uncommitted changes
git stash list               # Show all stashes
git stash apply              # Apply last stash without removing it
git stash pop                # Apply last stash and remove it from the list
git stash drop               # Remove a specific stash
```

### ðŸ”¹ Git Configuration

```bash
git config --global user.name "Your Name" # Set global username
git config --global user.email "you@example.com" # Set global email
git config --global core.editor "vim" # Set default editor
git config --global --list  # Show all global configurations
```

## ðŸ“Œ Conclusion

These Git commands cover the most common workflows and help developers effectively manage version control.
